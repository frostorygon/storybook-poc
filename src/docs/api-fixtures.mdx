## 2. Action Submissions (Success States)

Holding or unholding the card submits data via POST. The UI confirms these actions based purely on the successful 200/204 response.

**Hold Endpoint (Action):** `POST /api/v1/cards/{cardId}/hold`
**Unhold Endpoint (Action):** `POST /api/v1/cards/{cardId}/unhold`

*(No payload required for these actions other than auth headers)*
