# Error Screen

Catch-all error screen for all failure scenarios in this feature.

## States

| State              | Story Name                 | Description                              |
|--------------------|----------------------------|------------------------------------------|
| Generic failure    | `Default - Generic`        | Unspecified backend error                |
| Request timeout    | `Default - Timeout`        | Request took too long                    |
| Session expired    | `Default - Session Expired`| Auth token expired, user must re-login   |

## Design Decision

All error scenarios use the same screen with different copy and CTA behavior,
rather than separate error screens. This keeps the error surface minimal and
makes it easier to maintain consistency.

## Fixtures

Each fixture maps to a different error type. The `errorCode` field drives
which copy and CTA the screen renders.
