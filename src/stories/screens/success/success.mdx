# Success Screen

Confirmation screen after a successful hold or unhold action.

## States

| State           | Story Name         | Description                           |
|-----------------|--------------------|---------------------------------------|
| Card held       | `Default - Hold`   | Confirms card has been put on hold    |
| Card unheld     | `Default - Unhold` | Confirms card has been reactivated    |

## API Dependencies

None — this screen receives its data from the previous step via `feature-flow` state.
