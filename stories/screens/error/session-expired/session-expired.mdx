# Session Expired Error Screen

**A smart component demonstrating domain-specific error handling.**

The `session-expired-error-screen` is a **Smart Component**. It represents an enterprise pattern where a domain-specific action (Authentication Redirect) is encapsulated inside the screen itself.

Unlike generic errors (which also have their own self-contained components like `generic-error-screen` and `timeout-error-screen`), this component additionally manages specialized auth-redirect logic — demonstrating how Smart Components handle unique domain behavior beyond just differing text.

## Behavior & Responsibility

- Displays a hard-coded Session Expired message.
- The "Go to Login" button natively triggers an `auth-redirect` event.
- In a real application, this event would be captured by a top-level router or authentication service to perform an OAuth/OIDC redirect.

## Usage Example

```js
import { html, LitElement } from 'lit';
import { SessionExpiredErrorScreen } from '@scope/session-expired-error-screen';

class MyFeature extends LitElement {
  static get scopedElements() {
    return {
      'session-expired-error-screen': SessionExpiredErrorScreen,
    };
  }

  render() {
    return html\`
      <session-expired-error-screen
        @auth-redirect="\${this.handleAuthRedirect}">
      </session-expired-error-screen>
    \`;
  }
  
  handleAuthRedirect() {
    // Escalate to auth service
    window.location.href = '/login';
  }
}
```
